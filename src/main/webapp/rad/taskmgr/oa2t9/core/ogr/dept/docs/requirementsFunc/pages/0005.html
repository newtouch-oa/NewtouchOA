功能描述：

1.部门：

             增加：当用户第一次进到用户管理的时候，首先要对其进行添加部门（填入相关信息）。相关信息添加后就会在部门管理页面的左侧出现一个树状列表，在这里会显示用户所添加后的部门列表。

             修改：当用户点击树状列表部门的时候，在页面的右侧就会出现当前对应部门的所有信息，这时候用户就可以对其的信息进行修改。

             删除：在页面的下方有个删除的按钮，点击它可以删除当前页面显示的部门，如果当前部门还有下级部门，一起删除。

             对部门文件的导入和导出

2.公共自定义组：

             增加：在部门列表的下方有个公共自定义组按钮，点击它可以就会显示公共自定义组的列表，如果没有任何公共自定义组，这是就可以新建一个公共自定义组（点击新建公共用户组）。点击后就会出现公共用户组所要填写的信息。

             编辑：在管理公共用户组列表中有个编辑的按钮，点击它可以对点击当前的信息进行修改。

             删除：管理公共用户组列表中同样有个删除的按钮，点击它可以对点击当前的管理公共用户组信息进行删除。

重难点：

  1. 假如有5级部门，在新建部门的列表中的上级部门的下拉框里面的内容怎么去显示。
  2. 在点击部门列表的时候，它会把这个部门的信息显示在页面上，在信息当中有个上级部门的下拉框，它会显示当前部门的上级部门。
  3. 这个部门是不是一个主键
  4. 删除的时候应该删除的是一个主键，部门的下级部门之间是一个什么样的关系
  5. 在新建部门、编辑部门等都有一个添加的功能按钮，点击它的时候会弹出个对话框，里面的功能是怎么实现的

页面的布局：

           在对部门进行编辑的时候，在编辑的下面有个当前部门的相关操作，这个地方有些不好让人发现（刚开始没发现）（跟编辑有点脱节）
