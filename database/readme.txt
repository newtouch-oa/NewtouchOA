establish : 到一定阶段的完整建库脚本，存放在该目录
migration : 数据迁移相关脚本，存放在该目录
package : 本项目使用包(package)创建脚本
source  : 本项目使用到的其他比较复杂的可执行对象的创建脚本，包括过程(procedure)、方法(function)、触发器(trigger)等。
	  类型(type)、视图(view)、索引(index)、序列(sequence)、同义词(synonyms)等因为比较简单，一般就存放在下面的积累更新脚本中(update_pack.sql)。
	  这个原则可以灵活掌握。
script  : 并非升级用，但有必要保存的脚本，存放在该目录。例如一些复杂查询语句、数据更新脚本、调试用的脚本等。
update  : 对外发布的升级脚本，一般内部再按发布日期分为各个子目录。

update_pack.sql : 项目积累更新脚本，内部按照增加日期列出。
                  应当在编码时同步维护该脚本。
                  一般希望能达到，运行该脚本即可更新到本项目最新状态。
                  对外发布的升级脚本，即以此脚本为蓝本。


对外发布脚本，一般是等需要送测时，即按照日期在update目录下创建新的送测脚本目录。
然后从积累更新脚本(update_pack.sql)把最近一次发布到现在的内容复制到送测脚本目录下，
并按照需要复制相应的package、source等代码。
